# USDT Staking Vault ‚Äì EDUChain

## üéØ Goals

1. **Increase Stablecoin TVL on EDUChain**
   - Attract USDT/USDC from Arbitrum and other chains.
   - Users mint OCUSD on EDUChain using bridged stables.
   - Consider restricting withdrawals of OCUSD to EDUChain only (to encourage liquidity retention).

2. **Deliver Sustainable Yield for Stablecoin Holders**
   - Deploy bridged USDT/USDC into off-chain or L2 strategies (e.g. Ethena, USDX).
   - Return yield manually via REWARDER_ROLE calls to on-chain vaults.

---

## üìù Product Concept

Users deposit USDT/USDC ‚Üí mint **OCUSD** ‚Üí stake OCUSD ‚Üí earn yield from externally deployed strategies.

This follows a similar model to [Ethena](https://www.notion.so/Ethena-Fork-EDUChain-1fa4928fd54f80e88bf7d39fd40af362?pvs=21) or USDX.

---

## üîë Key Features

- **ERC20 Stablecoin: OCUSD**
  - Minted from USDT or USDC
  - Redeemable via cooldown process

- **ERC4626 Vault: sOCUSD**
  - Yield-bearing token (capital appreciating)
  - Requires cooldown + `claim()` flow

- **Manual Yield Streaming**
  - Yield streamed by multisig/backend using `transferInRewards()`
  - Controlled by `REWARDER_ROLE`

- **Governance Parameters**
  - Fee configuration, minter roles, pause, blacklists

## üîß Smart Contract Requirements

### ERC20 Contract (OCUSD)

**P0 (Must Have):**
- Mintable only by a permissioned address
- `isMinter` role management
- Burn/redemption function
- Governance params: blacklist (optional), pause

**P1 (Nice to Have):**
- OFT-compatible (LayerZero)

---

### ERC20 Minting/Sales Contract

**P0 (Must Have):**
- `buy()`, `claim()`, `redeem()` / `cooldown()` methods
- Fee config
- Settable cooldown time
- List of supported stablecoins

**Examples:**
- USDX Sale Contract: `0xb45c42Fbf8AF8Df5A1fa080A351E9B2F8e0a56D1`

---

### ERC4626 Vault Contract (sOCUSD / sEDU)

**P0 (Must Have):**
- Cooldown initiation (`cooldownAssets()` or `cooldownShares()`)
- Claim after cooldown (`unstake()` or `claim()`)
- `previewWithdraw()` or `getAmountByShares()`
- `REWARDER_ROLE` with `transferInRewards()`
- Configurable governance params

## üîÑ User Flow

### USDT/USDC Users

1. Deposit ‚Üí receive OCUSD
2. Stake OCUSD ‚Üí receive sOCUSD
3. Earn yield via vaults
4. Initiate cooldown ‚Üí call `claim()` ‚Üí withdraw

### EDU Users

1. Stake EDU ‚Üí receive sEDU (capital-appreciating LST)
2. Use sEDU in recursive DeFi
3. Unstake after cooldown to exit
